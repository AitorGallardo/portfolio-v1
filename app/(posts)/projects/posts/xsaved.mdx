# The problem

X (Twitter) gives you one flat, unsearchable list of bookmarks. Save a few hundred tweets and finding anything again is hopeless — there's no search, no tags, no folders, and if a tweet gets deleted, your bookmark is gone with it.

XSaved fixes that. It turns thousands of bookmarks into a **local-first, searchable, AI-organized library** — no accounts required, with your data living in your own browser.

# One product, three surfaces

XSaved started as a Chrome extension and is growing into a small native suite. The apps share one design system, data model, and bookmark-extraction core.

## 1. Chrome Extension — shipped &amp; live

The flagship: a Chrome MV3 extension (TypeScript + React 19) that captures your bookmarks straight from x.com and stores them locally, then layers search, tagging, folders, notes, and on-device AI topic clustering on top.

A native macOS app with a translucent sidebar, a dashboard of headline stats, an image-forward grid, and a native inspector for detail. It shares the iOS design system and models, and the same off-screen `WKWebView` extraction approach for reading bookmarks and running live actions.

# Under the hood

XSaved is small in surface area but dense in interesting engineering problems:

- **Local-first by design** — every saved tweet lives in the user's browser first (Dexie / IndexedDB). The cloud backend exists only for auth, billing, and AI taxonomy — never as the primary store for personal data.
- **Sub-50ms search over 50K+ bookmarks** — a custom IndexedDB query pipeline with a token-aware parser, multi-entry indexes, and a hot/warm/cold cache layer, with no server round-trip.
- **On-device AI** — a full embeddings → hierarchical clustering → topic-naming pipeline runs entirely in the browser via ONNX Runtime Web and Transformers.js, in an offscreen document so the UI never blocks.
- **Surviving X's frontend churn** — early request interception captures the bookmarks API response before page scripts run, isolating the product from cosmetic DOM changes on x.com.
- **A typed, serverless backend** — a single Hono app on Cloudflare Workers fronting Neon Postgres via Drizzle ORM, with JWT auth, Stripe subscriptions, and an Anthropic Claude-powered taxonomy workflow for premium AI features.
